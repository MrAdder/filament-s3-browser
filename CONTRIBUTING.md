# Contributing

Thanks for contributing to Filament S3 Browser.

## Development setup

This package is developed as a Laravel library, so `composer.lock` is intentionally not tracked. Install dependencies with:

```bash
composer update
```

Run the quality checks before opening a pull request:

```bash
vendor/bin/pint --test
vendor/bin/phpstan analyse --no-progress
vendor/bin/pest
```

## Pull request guidelines

Please keep pull requests focused and include tests for any behavior changes.

If your change affects the public API, configuration, or user-facing behavior, update the documentation in `README.md` as part of the same pull request.

Do not manually edit `CHANGELOG.md`. Release entries are generated by release-please from commit history.

## Commit messages

This repository uses Conventional Commits so release automation can determine the correct next version.

Examples:

- `fix: handle nested directory breadcrumbs correctly`
- `feat: add preview support for signed PDF routes`
- `feat!: rename config key for navigation grouping`

Use `!` or a `BREAKING CHANGE:` footer when a change is not backwards compatible.

## Reporting issues

Use the GitHub issue templates for bugs and feature requests.

For security reports, follow the guidance in `SECURITY.md` and avoid creating a public issue.
