# Security Policy

## Supported versions

| Version | Supported |
| --- | --- |
| `main` | Yes |
| Latest tagged release | Yes, when available |
| Older tagged releases | No |

## Reporting a vulnerability

Please do not open public GitHub issues for security vulnerabilities.

Use GitHub's private vulnerability reporting flow when it is available for the repository:

<https://github.com/mradder/filament-s3-browser/security/advisories/new>

If private vulnerability reporting is unavailable, contact the repository owner privately through GitHub and include:

- A description of the issue and the likely impact
- The package version and framework versions involved
- The affected filesystem disk or driver
- Clear reproduction steps or a proof of concept

We will aim to acknowledge valid reports within five business days and will coordinate a fix and disclosure timeline from there.
